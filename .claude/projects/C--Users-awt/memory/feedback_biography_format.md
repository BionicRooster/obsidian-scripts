User explicitly praised `Helen Cordes.md` as the ideal model for people notes.

**Key elements (in order):**
- Rich **frontmatter**: tags, created, modified dates, nav property pointing to MOC
- Header with `[[15 - People]] | [[MOC - relevant]]` breadcrumb
- **Biography** paragraph summarizing the person in their own words
- **Career History** section with named subsections per role, bullet details under each
- **Books & Publications** as a markdown table (Title, Year, Publisher, Notes)
- **Projects** section for major ongoing work
- **Community Involvement** section
- **Selected Articles** as a markdown table with links (where applicable)
- **Personal Life** section (birth, family, death if applicable)
- **Related Notes** organized by themed sub-headings

**Research standard:** For public figures, research from 10+ web sources before writing. File location: `C:\Users\awt\Sync\Obsidian\15 - People\<Full Name>.md`

**REQUIRED: All three resource types must be used for every person file (2026-05-30):**
1. **Vault search** — Grep for the person's name across all vault files; read any GCCMA/org meeting notes, contact files, or existing references
2. **Web search** — WebSearch + WebFetch for news coverage, nonprofit filings, obituaries, public records, org websites
3. **Email search** — eM Client local SQLite databases (all 11+ accounts); spawn Haiku subagent with the standard search script from `tools/emclient.md`

Do not skip any of the three. Missing one is a gap, not a shortcut.

**David Packard.md** is the second approved gold standard (same format confirmed valid).
